\n The Horizontal Pod Autoscaler automatically scales the number of Pods in a replication controller, deployment, replica set or stateful set based on observed CPU utilization (or, with custom metrics support, on some other application-supplied metrics).\n

\n
\n

Key Configuration Parameters

\n
    \n
  • minReplicas: The lower limit for the number of replicas to which the autoscaler can scale down.
    • \n
  • maxReplicas: The upper limit for the number of replicas to which the autoscaler can scale up.
    • \n
  • targetCPUUtilizationPercentage: The target average CPU utilization (represented as a percentage of requested CPU) over all the pods.
    • \n
\n
\n

Algorithm Details

\n

\n The Horizontal Pod Autoscaler is implemented as a control loop, with a period managed by the controller manager's --horizontal-pod-autoscaler-sync-period flag (default is 15 seconds).\n

\n

\n During each period, the controller manager queries the resource utilization against the metrics specified in each HorizontalPodAutoscaler definition. The controller manager obtains the metrics from either the resource metrics API (for per-pod resource metrics), or the custom metrics API (for all other metrics).\n

\n
